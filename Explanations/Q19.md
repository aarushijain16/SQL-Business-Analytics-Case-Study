## 🧠 Problem Statement
[Q19](https://github.com/aarushijain16/SQL-Business-Analytics-Case-Study/blob/main/Queries/Q19.sql). Analyze average order size in terms of quantity and revenue to understand purchasing behavior and identify opportunities for upselling and cross-selling.

## 🎯 Objective
To evaluate customer purchasing patterns by analyzing order size and identifying opportunities to increase basket value.

## 🛠️ Approach
- Aggregated order-level metrics:
  - Total items per order using SUM(quantity)
  - Total order value using SUM(revenue)
- Used a Common Table Expression (CTE) to structure intermediate results
- Calculated:
  - Average items per order
  - Average order value
- Segmented orders into basket types (Single Item, Small Basket, Big Basket) using CASE()
- Counted number of orders in each basket category

## ⚙️ Why This Approach?
Order-level aggregation is essential to understand purchasing behavior at the transaction level before deriving overall metrics.  
AVG() is used to compute average basket size and value.  
CASE() is used to classify orders into meaningful segments, enabling better interpretation of customer buying patterns.

## 💡 Business Insight
This analysis helps businesses:
- Identify opportunities for upselling and cross-selling  
- Understand typical basket size and spending behavior  
- Design strategies to increase average order value (AOV)  
- Encourage customers to move from single-item to multi-item purchases
