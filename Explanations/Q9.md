## 🧠 Problem Statement
[Q9](https://github.com/aarushijain16/SQL-Business-Analytics-Case-Study/blob/main/Queries/Q9.sql). Segment customers using Recency, Frequency, and Monetary (RFM) analysis to identify high-value, loyal, and at-risk customers.

## 🎯 Objective
To categorize customers based on their purchasing behavior and value contribution, enabling targeted retention and engagement strategies.

## 🛠️ Approach
- Aggregated customer-level metrics:
  - Last order date using MAX()
  - Total number of orders using COUNT()
  - Total revenue using SUM()
- Used a Common Table Expression (CTE) to structure intermediate results
- Calculated recency as the difference between the dataset’s latest order date and each customer’s last order
- Applied rule-based segmentation using CASE() to classify customers into different groups

## ⚙️ Why This Function?
MAX() OVER() is used to retrieve the latest order date in the dataset, which acts as a reference point for calculating recency.  
Using CURRENT_DATE was avoided because the dataset is historical, and using the system date would lead to incorrect recency values.  
CASE() is used to classify customers into segments based on defined thresholds for frequency, monetary value, and recency.

## 💡 Business Insight
This analysis helps businesses:
- Identify high-value customers for priority retention and personalized offers  
- Detect at-risk customers and take proactive re-engagement actions  
- Understand customer behavior patterns across value segments  
- Allocate marketing efforts more effectively based on customer importance
